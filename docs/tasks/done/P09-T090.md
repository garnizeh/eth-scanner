## Description

Perform a final end-to-end test of the ESP32 worker hardware interfacing with the Master API.

## Acceptance Criteria

- [x] ESP32 correctly requests a lease from the Master API (`POST /api/v1/jobs/lease`).
- [x] ESP32 Core 1 starts generating keys correctly for the assigned `prefix_28`.
- [x] ESP32 Core 0 periodic checkpointing via `PATCH /api/v1/jobs/{id}/checkpoint` successfully updates the Master API server.
- [x] ESP32 completes the job and sends results (`POST /api/v1/jobs/{id}/complete`).

## Implementation Notes

- Monitor serial output for "Lease successful, starting scan".
- Verify the Master API's SQLite database is updated with the ESP32's worker ID.

## Testing

- Serial monitoring and checking the Master API logs/database status.
