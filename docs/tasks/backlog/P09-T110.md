## Description

Perform final verification of all Master API endpoints with external tooling to ensure the HTTP server and database integration are bug-free.

## Acceptance Criteria

- [ ] `GET /health` returns 200 OK.
- [ ] `POST /api/v1/jobs/lease` returns a job JSON correctly.
- [ ] `PATCH /api/v1/jobs/{id}/checkpoint` updates the DB correctly.
- [ ] `POST /api/v1/jobs/{id}/complete` finishes the job correctly.
- [ ] `POST /api/v1/results` saves new results correctly.
- [ ] `GET /api/v1/stats` returns statistical data from the SQLite view.

## Implementation Notes

- Create a collection of `curl` commands or a Postman JSON export in `docs/api/tests/`.

## Testing

- Execute all test requests via Postman and verify the JSON responses/SQLite state.
